# Plan: Speed Up Navigation

## Problem
Navigating between pages (e.g., Home → Shop) is slow because:
1. **Middleware runs `supabase.auth.getUser()` on every navigation** — adds a network round-trip on each page change
2. **No `loading.tsx` files** — no visual feedback during transitions, users see blank screens
3. **Shop page fetches products server-side on every visit** — server component blocks rendering

## Solution

### 1. Optimize Middleware (`middleware.ts`)
- Only run Supabase auth check for `/admin` routes (the only routes that need it)
- Skip auth for public routes (`/`, `/shop`, `/track`, `/checkout`, `/product/*`)
- This eliminates the unnecessary network call on every navigation

### 2. Add `loading.tsx` files for instant feedback
- `src/app/loading.tsx` — Home page skeleton
- `src/app/shop/loading.tsx` — Shop page skeleton
- `src/app/track/loading.tsx` — Track page skeleton
- `src/app/checkout/loading.tsx` — Checkout page skeleton
- `src/app/product/[id]/loading.tsx` — Product page skeleton
- These show immediately during navigation instead of a blank screen

### 3. Add prefetching hints to Navbar Links
- The `<Link>` component already prefetches by default in Next.js, but we can add `prefetch={true}` explicitly to ensure it works

## Files to modify
